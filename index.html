The Problem

Research is slow, fragmented, and expensive.

Information overload

Research topics spread across papers, news, encyclopedic context, and live web sources.

Shallow AI tools

Most assistants stop after one answer instead of expanding into new unexplored subtopics.

Cost barrier

Premium deep-research products are often priced beyond student teams and solo builders.

System Architecture

Modular by design

Each module owns one job and feeds the central orchestration loop.

Core Flow

  1. Input topic
  2. Search multiple engines
  3. Extract full content
  4. Summarize findings
  5. Discover next subtopics
  6. Compile final report

Backend Modules

  • main.py / agent.py
  • duckduckgo.py / wikipedia_search.py
  • arxiv_search.py / news_search.py
  • web_scraper.py / content_extractor.py
  • summarizer.py / subtopic_generator.py
  • report_generator.py / config.py

Technology Stack

  • Python 3.8+
  • DuckDuckGo search
  • Wikipedia API
  • ArXiv API
  • Google News RSS
  • BART-Large-CNN summarization

Key Features

Autonomy, iteration, intelligence

Endless Loop Mode

Raise iterations toward continuous autonomous exploration.

Multi-Engine Search

Blend current web results, background knowledge, papers, and news.

AI Summarization

Condense long source content into focused, report-ready outputs.

Curiosity Engine

Generate new research angles from observed gaps in the evidence.

Auto Report Writer

Assemble structured documents with sections, summaries, and citations.

Private by Design

Runs locally in the original concept so sensitive topics stay under user control.
